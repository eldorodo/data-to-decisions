Decision Guide

High cardinality (>20 categories)? → Target Encoding or Hash Encoding to avoid dimensionality explosion
Ordinal/ordered categories? → Ordinal Encoding (e.g., Low=1, Med=2, High=3)
Low cardinality (<10 categories)? → One-Hot Encoding for tree models and linear models
Repeated CV folds / leakage risk? → Leave-One-Out encoding with proper cross-validation splits
Very high cardinality (e.g., zip codes)? → Hash Encoding with fixed output size

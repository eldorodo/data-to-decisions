RAG: Retrieval-Augmented Generation

1 User Query →
2 Embed Query (dense vector) →
3 Vector DB Search (cosine/ANN) →
4 Retrieve Top-k Documents →
5 [Context + Query] → LLM →
6 Grounded Answer
RAG reduces hallucinations by grounding LLM responses in retrieved factual documents — no model retraining needed.
